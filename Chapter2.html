Chapter 2: Variables and Data Types - Summary

1. Input and Output Methods

  • alert() — Displays a message box with text or variable values.
  • confirm() — Shows a dialog box with “OK” and “Cancel.” Returns true if OK is clicked, false otherwise.
  • prompt() — Displays an input box for user values, optionally with a default value. Returns null if canceled.
  • console.log() — Outputs information to the browser console for debugging.
  • document.write() — Displays text or variables directly on the web page.

2. Variables

  • A variable is a named storage space for data values.
  • Acts as a “label” attached to memory where the value is stored.
  • Each variable must have a unique name to avoid conflicts.

3. Naming Rules

  • Cannot start with a number or contain spaces.
  • Case-sensitive — num, Num, and NUM are different.
  • Common styles:
    • snake_case: current_year
    • camelCase: currentYear
  • Reserved keywords (e.g., let, const) cannot be used as variable names.

4. Declaring and Assigning Variables

  • Use let or const to declare variables.
  • let allows reassignment; const defines constants that cannot be reassigned.
  • Examples:
    • let x = 10;
    • const pi = 3.14;

5. Why Use Constants?

  • Constants store fixed values that don’t change, such as current year or DOM elements.
  • Improves readability and makes code easier to update.
  • Example:
    const currentYear = 2025;
    let age = currentYear - birthYear + 1;

6. var, let, and const

  • var — older keyword (can be redeclared and reassigned).
  • let — cannot be redeclared, but can be reassigned.
  • const — cannot be redeclared or reassigned.

7. Data Types in JavaScript

  • Two main categories:
    • Primitive types: number, string, boolean, undefined, null, symbol.
    • Objects: arrays, functions, and user-defined objects.
  • Check type using typeof():
    typeof("Hello") // 'string'

8. Numbers and Strings

  • JavaScript uses a single number type for both integers and floats.
  • Strings are enclosed in single (' ') or double (" ") quotes.
  • Escape special characters with backslashes (\), e.g. \n for new line.
  • Use template literals with backticks (`) for easy variable insertion:
    console.log(`${name}, welcome!`);

9. Booleans, Truthy & Falsy

  • Boolean values: true or false.
  • Falsy values: 0, "", NaN, undefined, null.
  • All other values are considered truthy.

10. undefined and null

  • undefined — variable declared but not assigned.
  • null — intentionally empty value.
  • Example:
    let x;
    let y = null;

11. Arrays

  • Used to store multiple values in one variable.
  • Defined with square brackets [ ] and separated by commas.
  • Access elements by index (starting from 0): colors[1].
  • Get array length: colors.length.

12. Symbols

  • Introduced in ES6 — a primitive data type ensuring unique identifiers.
  • Used as object property keys to avoid naming conflicts.
  • Example:
    let id = Symbol();
    const member = { [id]: 123 };

13. Type Conversion

  • JavaScript automatically converts types when performing operations (implicit conversion).
  • Use built-in functions for explicit conversion:
    • Number("10") → 10
    • parseInt("10.5") → 10
    • parseFloat("10.5") → 10.5
    • String(true) → "true"
    • Boolean(0) → false

14. Practice Example – Fahrenheit to Celsius Converter

  • Formula: C = (F - 32) / 1.8
  • Use parseFloat() to ensure decimal precision.
  • Use toFixed() to limit decimal places in results.
  • Example: 
          let f = parseFloat(prompt("Enter temperature in Fahrenheit"));
      let c = ((f - 32) / 1.8).toFixed(1);
      alert(`Fahrenheit ${f}° equals Celsius ${c}°.`);
